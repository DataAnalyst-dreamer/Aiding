## 1. Executive Summary

Aiding(아이딩)은 0~36개월 첫째 아이를 키우는 30대 초반 맞벌이 부부를 위한 AI 기반 육아용품 큐레이션 앱이다.

핵심 해결 과제는 "육아용품 결정 장애"이다. 첫째 아이를 키우는 초보 부모는 매달 바뀌는 발달 단계에 맞춰 어떤 용품을 언제 사야 하는지 판단하지 못한다. 기존 해결책(맘카페, 블로그, 지인 추천)은 정보의 신뢰성이 낮고, 상업적 바이어스가 크며, 부모의 개별 상황(아이 월령, 예산, 주거 환경)을 반영하지 못한다.

Aiding은 Gemini AI를 활용하여 아이의 월령에 맞는 육아용품을 자동 추천하고, 쿠팡파트너스·네이버 쇼핑 제휴 링크로 즉시 구매를 연결한다. 수익 모델은 프리미엄 구독(B2C)과 제휴 커머스 수수료(B2B)의 투트랙(Two-Track) 구조이다.

운영 구조는 "1인 창업자 + AI 8-Agent 자동화"를 전제로 설계한다. 창업자는 주 4~6시간만 투입하며, 8개 AI Agent가 개발·마케팅·CX·재무·QA·법무를 분담 처리하는 오케스트레이션 체제로 운영한다.

## 2. Problem Statement

### 2.1 핵심 문제 정의

첫째 아이를 키우는 초보 부모는 다음 세 가지 문제에 직면한다.

**정보 과부하와 신뢰성 부재**: 육아용품 관련 정보는 맘카페, 블로그, SNS에 넘쳐나지만, 대부분 개인 경험 또는 광고성 콘텐츠이다. 부모는 어떤 정보가 신뢰할 수 있는지 판단할 기준이 없다.

**월령별 필요 용품의 예측 불가**: 0~36개월은 발달 변화가 급격한 시기이다. 부모는 "다음 달에 무엇이 필요한지"를 사전에 알 수 없어, 필요한 시점에 급하게 검색하고 구매한다. 이 과정에서 불필요한 제품을 사거나, 필요한 제품을 늦게 구매하는 일이 반복된다.

**의사결정 시간의 부족**: 맞벌이 부부는 퇴근 후 제한된 시간 안에 육아용품을 조사하고 비교해야 한다. 한국보건사회연구원의 '2021년도 가족과 출산조사'에 따르면, 1인당 영유아 자녀 관련 월평균 지출은 60만 6천원이다. 이 금액의 상당 부분이 충분한 비교 없이 소비된다.

### 2.2 기존 대안의 한계

| 기존 대안 | 한계 |
|-----------|------|
| 맘카페 (네이버 맘카페, 맘스홀릭 등) | 정보 신뢰도 편차 큼, 광고성 후기 혼재, 내 아이 월령 맞춤 정보 필터링 불가 |
| 육아 앱 (베이비타임, 그로우 등) | 발달 정보 제공에 집중, 용품 추천과 구매 연결 부재 |
| 쇼핑몰 직접 검색 (쿠팡, 네이버) | 월령별 큐레이션 없음, 부모가 직접 검색어를 알아야 함, 정보 과부하 |
| SNS/인플루언서 추천 | 광고 협찬 제품 중심, 개별 상황 반영 불가, 과대 광고 위험 |

### 2.3 Aiding이 제공하는 해결 방식

"아이 생년월일 하나만 입력하면, AI가 매달 지금 필요한 육아용품을 골라주고, 최저가 구매 링크까지 연결한다."

기존 대안과의 핵심 차별점은 세 가지이다.

첫째, 공공기관 데이터 기반의 신뢰성. 보건복지부, 질병관리청, 대한소아청소년과학회 등 검증된 출처만 활용한다. 맘카페 후기나 인플루언서 추천은 배제한다.

둘째, 월령 기반 선제적 추천. 부모가 검색하기 전에, 아이의 발달 단계에 맞춰 "다음 달에 필요한 것"을 먼저 알려준다.

셋째, 추천과 구매의 즉시 연결. 추천 카드에서 한 번의 탭으로 쿠팡/네이버 최저가 구매 페이지로 이동할 수 있다.

## 3. Business Objectives & Success Metrics

### 3.1 사업 목표

**Phase 1 — 출시 3개월 차 (시장 검증)**:
- MAU 800명 확보
- 유료 구독 전환 80명 (전환율 10%)
- 월 수익: 구독 23만원 + 커머스 수수료 15만원 = 약 38만원

**Phase 2 — 출시 6개월 차 (손익분기 돌파, 월 100만원 목표)**:
- MAU 2,500명 확보
- 유료 구독 전환 250명 (전환율 10%)
- 월 수익: 구독 72만원 + 커머스 수수료 35만원 = 약 107만원

**Phase 3 — 출시 12개월 차 (안정화 및 확장)**:
- MAU 6,000명 확보
- 유료 구독 700명 + 브랜드 협찬 시작
- 월 수익: 약 280만원 이상

### 3.2 핵심 성과 지표 (KPIs)

| KPI | 정의 | 측정 주기 | 목표 (Phase 2) |

## 5. User Flow

```
앱 실행 (Splash)
    ↓
로그인 상태 확인
    ├─ 비로그인 → 소셜 로그인 (Google/Apple) → 온보딩 (아이 생년월일 입력) → 메인 쉘
    └─ 로그인 완료 → 메인 쉘
    
메인 쉘 (Bottom 4탭)
    ├─ [홈] 월령 맞춤 추천 카드
    ├─ [추천관] 큐레이션 쇼핑 (카테고리별 제품 탐색)
    ├─ [AI 상담] 육아 Q&A 챗
    └─ [성장 리포트] 주간 성장 마일스톤 + SNS 공유
    
구독 검사 (Paywall)
    ├─ 무료 사용자: 기본 추천 + AI 상담 3회/일 제한
    └─ 프리미엄 콘텐츠 또는 상담 초과 시:
        → 구독 결제 유도 (7일 Free Trial → 월 2,900원)
        → 결제 성공 → 프리미엄 콘텐츠 잠금 해제
    
제품 카드 탭 → 제휴 링크 인앱 브라우저 이동 (쿠팡/네이버)
```

## 7. Functional Requirements

### FR-01: 온보딩

| 항목 | 요구사항 |
|------|----------|
| 입력 정보 | 아이 생년월일 (필수), 아이 이름 또는 별명 (선택) |
| 처리 | 생년월일 → 현재 월령 자동 계산 → Firestore users 컬렉션 저장 |
| 결과 | 입력 즉시 홈 화면에 월령 맞춤 추천 카드 표시 |
| 유효성 검증 | 미래 날짜 차단, 36개월 초과 시 서비스 대상 외 안내 |
| 소요 시간 | 전체 온보딩 60초 이내 완료 |

### FR-02: 월령 맞춤 AI 추천

| 항목 | 요구사항 |
|------|----------|
| 추천 로직 | 월령 기반 → Gemini API → 카테고리별 추천 용품 생성 |
| 카테고리 | 수유/이유식, 의류, 위생/건강, 놀이/발달, 외출, 안전/가구 (월령별 상이) |
| 카드 구성 | 카테고리명, 추천 이유 (1~2문장), 제휴 링크 버튼, 가격 범위 |
| 갱신 주기 | 월령 변경 시 자동 (매월 1회) + 수동 새로고침 |
| 프리미엄 전용 | 일부 카테고리는 프리미엄 구독자만 잠금 해제 |

### FR-03: 프리미엄 구독 (Paywall)

| 항목 | 요구사항 |
|------|----------|
| 가격 | 월 2,900원 |
| Free Trial | 7일 무료 체험 후 자동 결제 전환 |
| 프리미엄 혜택 | AI 상담 무제한, 프리미엄 큐레이션 잠금 해제, 성장 리포트 전체 열람 |
| 결제 연동 | RevenueCat (Apple IAP + Google Play Billing 통합 관리) |
| 해지 방어 | 해지 시도 시 Winback Offer 제시 (예: 1개월 50% 할인) |
| 구독 상태 | Firestore users 컬렉션의 isPremium 필드로 관리 |

### FR-04: 선제적 푸시 알림

| 항목 | 요구사항 |
|------|----------|
| 발송 기준 | 월령 변경 1~2주 전, 다음 단계 필요 용품 사전 안내 |
| 발송 시간 | 평일 오후 7~8시 (퇴근 후 육아 시간대) |
| 빈도 제한 | 주 2회 이하, 사용자 빈도 조절 가능 |

### FR-05: 제휴 커머스 링크

| 항목 | 요구사항 |
|------|----------|

| 제휴사 | 쿠팡파트너스 (우선), 네이버 쇼핑 (보조) |
| 링크 유효성 | 주 1회 자동 점검, 만료/품절 시 비활성 처리 |
| 투명성 | "이 링크를 통해 구매 시 Aiding에 소정의 수수료가 지급됩니다" 명시 |

### FR-06: AI 챗 (육아 Q&A)

## 9. Revenue Model & Unit Economics

### 9.1 투트랙(Two-Track) 수익 구조

일반적인 커머스 앱과 달리, Aiding은 프리미엄 구독(B2C)과 제휴 커머스(B2B)를 동시 운영하여 수익 안정성을 확보한다.

| 수익원 | 유형 | 구조 | 예상 단가 |
|--------|------|------|-----------|
| **프리미엄 구독** (핵심) | B2C | 월 2,900원, 7일 Free Trial, RevenueCat 연동 | 인당 월 2,900원 |
| **제휴 커머스** (보조) | B2B | 쿠팡파트너스 구매 수수료 + 네이버 클릭 수수료 | 구매금액의 3~4% / 클릭당 50~150원 |

투트랙 구조의 이점: 구독 수익은 예측 가능한 반복 매출(MRR)을 제공하고, 제휴 수수료는 무료 사용자로부터도 수익을 발생시켜 전환 전 단계에서도 수익 기반을 유지한다.

## 10. Design System & Theming

### 10.1 컬러 팔레트

Aiding은 Light Mode와 Dark Mode를 모두 지원한다. 사용자는 앱 설정에서 Light / Dark / System(기기 설정 따르기) 중 선택할 수 있다.

**Light Mode**

| 역할 | 색상 | HEX | 용도 |
|------|------|-----|------|

| Background | 웜 화이트 | `#FAFAF8` | 앱 전체 배경 |
| Surface | 화이트 | `#FFFFFF` | 카드, 시트, 다이얼로그 배경 |
| Accent | 코랄 | `#FF6B6B` | 강조, 면책 조항, 뱃지, Paywall CTA |
| Text Primary | 다크 그레이 | `#1A1D20` | 본문 텍스트 |
| Text Secondary | 미디엄 그레이 | `#4A4D52` | 보조 텍스트, 캡션 |
| Text Tertiary | 라이트 그레이 | `#9E9E9E` | 플레이스홀더, 비활성 텍스트 |
| Divider | 라이트 그레이 | `#E0E0E0` | 구분선 |

**Dark Mode**

|------|------|-----|------|

| Divider | 다크 그레이 | `#2C2C2C` | 구분선 |

### 10.2 테마 구현 원칙

- Material 3 `ColorScheme.fromSeed`를 사용하여 민트 그린(seedColor)으로부터 Light/Dark 팔레트를 자동 생성한다.
- 코랄(Accent)은 면책 조항, Paywall CTA 등 "주의 환기"가 필요한 요소에만 제한적으로 사용한다.
- 다크 모드에서 Primary/Accent 색상은 밝기를 약간 올려 다크 배경 대비 충분한 명도 대비(contrast ratio 4.5:1 이상)를 확보한다.
- Provider(ChangeNotifier)로 테마 상태를 관리하며, `ThemeMode.system`을 기본값으로 사용한다.
- 앱 설정 화면에서 사용자가 Light / Dark / System을 직접 선택할 수 있다.

### 10.3 타이포그래피

- 기본 서체: Noto Sans KR (google_fonts 패키지 활용)
- 면책 조항/법적 문구: 코랄(Accent) 색상, 12pt, Bold

### 10.4 면책 조항 디자인 규칙

AI 챗, 매거진 콘텐츠, 추천 카드에 표시되는 면책 조항은 아래 규칙을 따른다:
- **배경**: Light Mode에서 `#FFF0F0` (연한 붉은색), Dark Mode에서 `#3D1F1F` (어두운 붉은색)
- **텍스트**: 코랄 Accent 색상, Bold
- **위치**: 해당 콘텐츠 하단에 하드코딩 (동적 제거 불가)
- **문구**: "본 콘텐츠는 참고용 정보이며, 의학적 진단을 대체하지 않습니다. 이상 증상 시 반드시 전문의 진료를 받으세요."

## 11. Operational Model — AI 8-Agent 기반 자동화

### 11.1 운영 구조 원칙

Aiding은 1인 창업자가 풀타임 직장을 병행하면서 운영하는 구조이다. 8개의 AI Agent가 실무를 분담 처리하는 오케스트레이션(Orchestration) 체제를 구축하며, 모든 자동화에 **"AI 실행 → 사람 승인 (Human-in-the-Loop)"** 원칙을 적용한다.

### 11.2 8-Agent 조직 구조

| 역할 | Agent | 담당 업무 | 자동화율 |
|------|-------|-----------|---------|
| **CEO** | 인간 창업자 | 전략 방향, 최종 승인, 파트너십 | 0% (직접) |
| **Dev** | 개발 에이전트 | 코드 작성, 버그 수정 (Gemini in Firebase Studio) | 90% |
| **CX** | CX/VoC 에이전트 | 인앱 AI 챗 1차 응대, 불만 키워드 정량화 리포트 | 85% |
| **Data** | 전략/데이터 에이전트 | Looker Studio 기반 유저 행동 분석, 마일스톤 시각화 | 80% |
| **Growth** | 그로스 에이전트 | 맘카페/릴스용 콘텐츠 초안, FCM A/B 푸시 세팅 | 90% |
| **CFO** | 재무 에이전트 | RevenueCat 구독 + 제휴 수수료 vs API 비용 일일 정산 | 95% |
| **PM** | PM 에이전트 | VoC + Data 합성 → 주간 스프린트 목표(PRD) 기안 | 85% |
| **QA** | QA 에이전트 | 결제/로그인 핵심 플로우 봇 테스트 10회 검증 | 90% |
| **Legal** | 법무 에이전트 | 앱스토어 정책 변동 스크래핑, 개인정보방침 위반 경고 | 90% |

### 11.3 주간 CEO 승인 워크플로우 (주 4~6시간)

| 요일 | 활동 | 소요 시간 |
|------|------|-----------|
| **월~화** | PM 에이전트가 VoC/Data 기반으로 기능 기획안 기안 → CEO 승인 | 1시간 |
| **수~금** | Dev 에이전트 코드 작성, QA 에이전트 봇 테스트 (CEO 개입 불필요) | - |
| **토** | QA All Green 확인 → 앱스토어 빌드 배포 승인 | 0.5시간 |
| **토** | Growth 에이전트가 생성한 콘텐츠 3건 검수 및 업로드 승인 | 0.5시간 |
| **토** | 매거진 초안 팩트체크 및 발행 승인 | 1시간 |
| **일** | CFO 에이전트 주간 LTV-CAC 차트 열람, 주간 KPI 대시보드 확인 | 0.5시간 |
| **일** | 앱스토어 리뷰 중 AI 미처리 건 대응, 다음 주 우선순위 정리 | 1시간 |

### 11.4 AI Agent 운영 파이프라인

**① 콘텐츠 생성 파이프라인 (Aiding 매거진)**

```
[자동] Growth 에이전트 — 월령별 토픽 기반 초안 생성 (화이트리스트 소스만 참조)
  ↓
[자동] 톤 검증 (블로그 말투, 이모지, 과대 표현 자동 필터링)
  ↓
[자동] 면책 조항 자동 삽입
  ↓
[사람] CEO — 팩트체크 후 발행 승인
  ↓
[자동] Firestore contents 컬렉션 저장 → 앱 노출
```

**② 월령 맞춤 추천 파이프라인**

```
[자동] 매일 자정 — 월령 변경 대상 사용자 목록 추출
  ↓
[자동] Gemini API — 해당 월령 추천 카드 생성
  ↓
[자동] 쿠팡/네이버 API — 카테고리별 최적 제품 매칭 및 제휴 링크 생성
  ↓
[자동] Firestore products 컬렉션 업데이트
  ↓
[사람] CEO — 주 1회 추천 품질 샘플 점검 (10건 랜덤)
```

**③ 선제적 푸시 알림 파이프라인**

```
[자동] 월령 변경 1~2주 전 대상자 추출
  ↓
[자동] Growth 에이전트 — 알림 메시지 생성, FCM A/B 테스트 세팅
  ↓
[사람] CEO — 발송 전 메시지 확인 (일괄 승인)
  ↓
[자동] FCM 예약 발송 (평일 오후 7~8시)
```

**④ 재무 모니터링 파이프라인**

```
[자동] CFO 에이전트 — RevenueCat 구독 수익 + 제휴 수수료 일일 수집
[자동] CFO 에이전트 — Firebase/Gemini API 비용 대비 수익 계산
[자동] 비용이 월 예산 80% 도달 시 CEO에게 알림
  ↓
[사람] CEO — 주 1회 주간 재무 리포트 열람
```

### 11.5 AI Agent 안전장치

| 안전장치 | 내용 |
|----------|------|
| 콘텐츠 화이트리스트 | 참조 소스를 공공기관으로 제한 (맘카페, 블로그 금지) |
| 금지 표현 필터 | 의학적 확언, 과대 광고, 제품 효과 보장 표현 자동 차단 |
| 발행 전 인간 검수 | 매거진, 푸시 알림은 CEO 승인 필수 |
| AI 챗 에스컬레이션 | 의학적 질문, 응급 상황 감지 시 "전문의 진료 안내"로 즉시 전환 |
| 비용 알림 | Firebase/Gemini 비용 월 예산 80% 도달 시 자동 알림 |
| Legal 에이전트 경고 | 앱스토어 정책 변경, 개인정보 규정 위반 가능성 자동 감지 |

products/
  {productId}/
    - category: string
    - targetMonths: array<number>
    - title: string
    - description: string
    - affiliateLink: string
    - affiliateSource: string ("coupang" | "naver")
    - priceRange: string
    - isPremiumOnly: boolean
    - isActive: boolean
    - updatedAt: timestamp

contents/
  {contentId}/
    - title: string
    - summary: string
    - body: string
    - targetMonths: array<number>
    - sourceUrl: string
    - sourceGrade: string ("1급" | "2급")
